Theming

FlexUI's theming system allows you to define consistent colors, typography, spacing, and other design tokens that can be referenced throughout your JSON configurations using template variables.

Theme Structure

A FlexUI theme consists of four main categories:

{
  "version": "1.0",
  "theme": {
    "colors": {
      "primary": "#FF6B00",
      "secondary": "#1A1A2E",
      "background": "#FFFFFF",
      "surface": "#F8F9FA",
      "text": "#333333",
      "textSecondary": "#666666",
      "success": "#4CAF50",
      "warning": "#FF9800",
      "error": "#F44336",
      "transparent": "#00000000"
    },
    "typography": {
      "headingFont": "sans-serif-medium",
      "bodyFont": "sans-serif",
      "headingSize": 24,
      "bodySize": 14,
      "captionSize": 12,
      "lineHeight": 1.4
    },
    "spacing": {
      "xs": 4,
      "sm": 8,
      "md": 16,
      "lg": 24,
      "xl": 32,
      "xxl": 48
    },
    "borderRadius": {
      "none": 0,
      "sm": 4,
      "md": 8,
      "lg": 16,
      "xl": 24,
      "full": 999
    }
  },
  "root": {
    // Your component tree here
  }
}

Colors

Define your color palette using hex values or named colors. Colors can be referenced using {{colors.colorName}}:

Primary Colors
  • primary - Main brand color
  • secondary - Secondary accent color
  • background - Main background
  • surface - Card/surface background
Text Colors
  • text - Primary text color
  • textSecondary - Secondary text
  • textOnPrimary - Text on primary color
  • textOnSecondary - Text on secondary color
Status Colors
  • success - Success states
  • warning - Warning states
  • error - Error states
  • info - Informational states

Typography

Define font families, sizes, and other text properties:

PropertyTypeDescriptionExample
headingFontStringFont family for headings"sans-serif-medium"
bodyFontStringFont family for body text"sans-serif"
headingSizeIntDefault heading size (sp)24
bodySizeIntDefault body size (sp)14
captionSizeIntCaption/small text size (sp)12
lineHeightFloatLine height multiplier1.4
letterSpacingFloatLetter spacing (em)0.1

Spacing

Define consistent spacing values for margins, padding, and gaps:

"spacing": {
  "xs": 4,    // Extra small - 4dp
  "sm": 8,    // Small - 8dp
  "md": 16,   // Medium - 16dp (base unit)
  "lg": 24,   // Large - 24dp
  "xl": 32,   // Extra large - 32dp
  "xxl": 48   // Extra extra large - 48dp
}

Border Radius

Define corner radius values for consistent rounded corners:

"borderRadius": {
  "none": 0,   // No rounding
  "sm": 4,     // Small radius
  "md": 8,     // Medium radius
  "lg": 16,    // Large radius
  "xl": 24,    // Extra large radius
  "full": 999  // Fully rounded (pill shape)
}

Variable Resolution

Reference theme values using double curly braces: {{category.property}}

Basic Usage

{
  "type": "button",
  "props": {
    "text": "Get Started",
    "backgroundColor": "{{colors.primary}}",
    "textColor": "{{colors.textOnPrimary}}",
    "padding": "{{spacing.md}}",
    "cornerRadius": "{{borderRadius.md}}"
  }
}

Nested Properties

Access nested theme properties using dot notation:

{
  "type": "text",
  "props": {
    "text": "Welcome!",
    "textSize": "{{typography.headingSize}}",
    "fontFamily": "{{typography.headingFont}}",
    "textColor": "{{colors.text}}",
    "lineSpacing": "{{typography.lineHeight}}"
  }
}

Data Variables

Combine theme variables with data passed from your app:

{
  "type": "text",
  "props": {
    "text": "Hello, {{data.userName}}!",
    "textColor": "{{colors.primary}}",
    "textSize": "{{typography.headingSize}}"
  }
}

Resolution Order: Variables are resolved in this order: 1) Data variables, 2) Theme variables, 3) Left as literal text if no match found.

Setting Default Themes

Server-side Theme

Include theme configuration in your JSON response:

{
  "version": "1.0",
  "theme": {
    "colors": { /* your colors */ },
    "typography": { /* your typography */ }
  },
  "root": { /* your components */ }
}

Client-side Default Theme

Set a default theme during FlexUI initialization:

Java
// Create default theme
FlexTheme defaultTheme = FlexTheme.getDefault();

// Or create custom theme
Map<String, String> colors = new HashMap<>();
colors.put("primary", "#FF6B00");
colors.put("background", "#FFFFFF");

Map<String, Integer> spacing = new HashMap<>();
spacing.put("md", 16);
spacing.put("lg", 24);

FlexTheme customTheme = new FlexTheme(colors, new HashMap<>(), spacing, new HashMap<>());

// Initialize with theme
FlexConfig config = new FlexConfig.Builder("https://api.com/flexui")
    .defaultTheme(customTheme)
    .build();
    
FlexUI.init(context, config);
Kotlin
// Create default theme
val defaultTheme = FlexTheme.getDefault()

// Or create custom theme
val customTheme = FlexTheme(
    colors = mapOf(
        "primary" to "#FF6B00",
        "background" to "#FFFFFF"
    ),
    spacing = mapOf(
        "md" to 16,
        "lg" to 24
    )
)

// Initialize with theme
val config = FlexConfig.Builder("https://api.com/flexui")
    .defaultTheme(customTheme)
    .build()
    
FlexUI.init(this, config)

Runtime Theme Updates

Update the default theme at runtime:

// Update theme for dark mode
Map<String, String> darkColors = new HashMap<>();
darkColors.put("background", "#121212");
darkColors.put("text", "#FFFFFF");
darkColors.put("surface", "#1E1E1E");

FlexTheme darkTheme = new FlexTheme(darkColors, new HashMap<>(), new HashMap<>(), new HashMap<>());
FlexUI.setDefaultTheme(darkTheme);

// Refresh any cached configurations to use new theme
FlexUI.clearCache();

Theme Override Patterns

Component-specific Overrides

Override theme values for specific use cases:

{
  "type": "button",
  "props": {
    "text": "Danger Action",
    "backgroundColor": "{{colors.error}}",
    "textColor": "#FFFFFF",
    "padding": "{{spacing.md}}",
    "cornerRadius": "{{borderRadius.md}}"
  }
}

Calculated Values

Use theme values as base for calculations (server-side logic):

{
  "type": "container",
  "props": {
    "padding": 24,  // spacing.lg * 1.5
    "backgroundColor": "#FFE0CC",  // primary color with alpha
    "child": {
      "type": "text",
      "props": {
        "text": "Featured Content",
        "textColor": "{{colors.primary}}"
      }
    }
  }
}

Conditional Theming

Apply different themes based on data conditions (server-side logic):

{
  "type": "card",
  "props": {
    "backgroundColor": "{{data.isPremium ? colors.primary : colors.surface}}",
    "elevation": "{{data.isPremium ? 8 : 2}}",
    "child": {
      "type": "text",
      "props": {
        "text": "{{data.isPremium ? 'Premium User' : 'Standard User'}}",
        "textColor": "{{data.isPremium ? colors.textOnPrimary : colors.text}}"
      }
    }
  }
}

Best Practices

Consistent Naming
  • Use semantic names: primary, secondary instead of blue, red
  • Follow size conventions: xs, sm, md, lg, xl
  • Be consistent across platforms (iOS/Android)
Design Tokens
  • Define a base unit (usually 8dp) and build from it
  • Limit your color palette to essential colors
  • Use typography scale with clear hierarchy
  • Define border radius values that match your design system
Performance
  • Keep theme objects reasonably sized
  • Use theme variables instead of hardcoding values
  • Cache theme configurations when possible
  • Avoid complex nested theme structures
Accessibility
  • Ensure sufficient color contrast (4.5:1 for normal text)
  • Support dark mode with appropriate colors
  • Use accessible font sizes (14sp minimum)
  • Provide clear visual hierarchy

Advanced Theming

Theme Inheritance

Server-side configurations can inherit from base themes:

{
  "extends": "base_theme",
  "theme": {
    "colors": {
      "primary": "#FF6B00"  // Override just the primary color
    }
  }
}

Platform-specific Themes

Define different themes for different platforms:

{
  "theme": {
    "android": {
      "colors": {
        "primary": "#FF6B00"
      },
      "typography": {
        "bodyFont": "Roboto"
      }
    },
    "ios": {
      "colors": {
        "primary": "#007AFF"
      },
      "typography": {
        "bodyFont": "SF Pro"
      }
    }
  }
}

Dynamic Themes

Load different themes based on user preferences:

// Load theme based on user preference
String themeMode = userPreferences.getString("theme", "light");
String configUrl = "https://api.com/flexui/themes/" + themeMode + "/welcome_screen";
View screen = FlexUI.render(context, configUrl);
