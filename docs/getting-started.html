Getting Started

Get FlexUI running in your Android project in under 5 minutes.

Installation

Step 1: Add Dependency

Add FlexUI to your build.gradle file:

dependencies {
    implementation 'dev.flexui:flexui-android:1.0.0'
}

Minimum Requirements: Android SDK 21+ (Android 5.0), Java 8+

Step 2: Initialize FlexUI

Initialize FlexUI in your Application class or SDK initialization:

Java
public class MyApplication extends Application {
    @Override
    public void onCreate() {
        super.onCreate();
        
        // Simple initialization
        FlexUI.init(this, "https://your-api.com/flexui");
        
        // OR with configuration
        FlexConfig config = new FlexConfig.Builder("https://your-api.com/flexui")
            .cachePolicy(CachePolicy.CACHE_FIRST)
            .cacheTtlMinutes(60)
            .connectTimeoutSeconds(10)
            .enableDebugLogging(true)
            .build();
            
        FlexUI.init(this, config);
    }
}
Kotlin
class MyApplication : Application() {
    override fun onCreate() {
        super.onCreate()
        
        // Simple initialization
        FlexUI.init(this, "https://your-api.com/flexui")
        
        // OR with configuration
        val config = FlexConfig.Builder("https://your-api.com/flexui")
            .cachePolicy(CachePolicy.CACHE_FIRST)
            .cacheTtlMinutes(60)
            .connectTimeoutSeconds(10)
            .enableDebugLogging(true)
            .build()
            
        FlexUI.init(this, config)
    }
}

Step 2: Render in Your Activity

Java
public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        
        ViewGroup container = findViewById(R.id.container);
        
        // Synchronous render
        View welcomeScreen = FlexUI.render(this, "welcome_screen");
        if (welcomeScreen != null) {
            container.addView(welcomeScreen);
        }
        
        // OR Asynchronous render
        FlexUI.renderAsync(this, "welcome_screen", new HashMap<>(), 
            new FlexRenderCallback() {
                @Override
                public void onSuccess(View view) {
                    container.addView(view);
                }
                
                @Override
                public void onError(FlexError error) {
                    // Handle error
                    Log.e("FlexUI", "Render failed", error);
                }
            }
        );
    }
}
Kotlin
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        
        val container: ViewGroup = findViewById(R.id.container)
        
        // Synchronous render
        val welcomeScreen = FlexUI.render(this, "welcome_screen")
        welcomeScreen?.let { container.addView(it) }
        
        // OR Asynchronous render
        FlexUI.renderAsync(this, "welcome_screen", emptyMap(),
            object : FlexRenderCallback {
                override fun onSuccess(view: View) {
                    container.addView(view)
                }
                
                override fun onError(error: FlexError) {
                    // Handle error
                    Log.e("FlexUI", "Render failed", error)
                }
            }
        )
    }
}

That's it! Your app will now fetch the JSON configuration from your server and render it as native Android views.

Passing Data to Configurations

You can pass dynamic data to your JSON configurations:

// Pass user data to the configuration
Map<String, Object> data = new HashMap<>();
data.put("userName", "John Doe");
data.put("userPoints", 1250);
data.put("userLevel", "Gold");

View userProfile = FlexUI.render(this, "user_profile", data);

Then reference the data in your JSON using {{data.fieldName}}:

{
  "type": "column",
  "props": {
    "children": [
      {
        "type": "text",
        "props": {
          "text": "Welcome back, {{data.userName}}!"
        }
      },
      {
        "type": "text",
        "props": {
          "text": "You have {{data.userPoints}} points ({{data.userLevel}} tier)"
        }
      }
    ]
  }
}

Handling Actions

Register action handlers to respond to user interactions:

Java
// Register navigation handler
FlexUI.onAction("navigate", new FlexActionHandler() {
    @Override
    public void handle(FlexAction action) {
        String url = action.getString("url");
        // Navigate to the specified URL
        startActivity(new Intent(Intent.ACTION_VIEW, Uri.parse(url)));
    }
});

// Register custom callback handler
FlexUI.onEvent("button_clicked", new FlexActionHandler() {
    @Override
    public void handle(FlexAction action) {
        // Handle custom button click
        String buttonId = action.getString("buttonId");
        showToast("Button clicked: " + buttonId);
    }
});
Kotlin
// Register navigation handler
FlexUI.onAction("navigate") { action ->
    val url = action.getString("url")
    // Navigate to the specified URL
    startActivity(Intent(Intent.ACTION_VIEW, Uri.parse(url)))
}

// Register custom callback handler  
FlexUI.onEvent("button_clicked") { action ->
    // Handle custom button click
    val buttonId = action.getString("buttonId")
    showToast("Button clicked: $buttonId")
}

Common Issues

Network Security Policy: If you're testing with HTTP URLs on Android 9+, add android:usesCleartextTraffic="true" to your Application tag in AndroidManifest.xml (development only).

ProGuard/R8: FlexUI uses reflection for JSON parsing. Add these rules to your proguard-rules.pro:

-keep class dev.flexui.schema.** { *; }
-keep class dev.flexui.FlexError { *; }

Initialization Error: Make sure to call FlexUI.init() before any render calls, preferably in your Application's onCreate().
