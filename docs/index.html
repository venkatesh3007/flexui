Quick Example

Transform this JSON configuration:

{
  "type": "column",
  "props": {
    "padding": "{{spacing}}",
    "children": [
      {
        "type": "text",
        "props": {
          "text": "Welcome to FlexUI",
          "textSize": "{{headingSize}}",
          "textColor": "{{primaryColor}}",
          "fontWeight": "bold"
        }
      },
      {
        "type": "button",
        "props": {
          "text": "Get Started",
          "backgroundColor": "{{primaryColor}}",
          "textColor": "#FFFFFF",
          "cornerRadius": "{{borderRadius}}",
          "action": {
            "type": "navigate",
            "url": "/onboarding"
          }
        }
      }
    ]
  }
}

Into native Android views with just one line:

// Initialize once
FlexUI.init(context, "https://your-api.com/flexui");

// Render anywhere
View welcomeScreen = FlexUI.render(context, "welcome_screen");
container.addView(welcomeScreen);

Why FlexUI?

For SDK Developers
Ship dynamic UIs without forcing Compose/SwiftUI on your customers. Java-compatible API that works everywhere.
For Product Teams
Update UI layouts, themes, and flows server-side. No app store approval needed for UI changes.
For Performance
Tiny footprint (~200KB) with smart caching. Native rendering means 60fps performance.
For Integration
Drop-in replacement for LayoutInflater.inflate(). Minimal changes to existing codebases.

Platform Support

Android Ready
  • Min SDK 21 (Android 5.0)
  • Kotlin internals, Java-compatible API
  • View system (no Compose dependency)
  • Maven Central distribution
  • ~200KB footprint
iOS Coming Soon
  • Min iOS 13.0
  • Swift with Objective-C support
  • UIKit (no SwiftUI dependency)
  • Swift Package Manager + CocoaPods
  • ~150KB footprint
